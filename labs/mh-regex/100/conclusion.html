Failure to use well tested or reputable regex patterns carries a potential security risk, enabling attackers with input potential to cause a Denial of Serivice targetting a specific regex pattern. Developers will often copy regex patterns from online sources, unwittenly copying one with issues such as nested repeition. Node.js open-source packages have had issues in the past with evilregex patterns, some of these packages may use a common vulnerable library as well. Testing regex patterns with specific static or dynamnic anaylsis tools is a good idea if you forsee the pattern in a crtiical or public role, as security is important.
